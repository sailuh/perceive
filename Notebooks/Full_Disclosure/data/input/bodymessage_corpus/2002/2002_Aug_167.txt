Under some circumstances, an invalid request may result in a allocated memory being freed twice.  This can potentially result in the execution of arbitrary code.  This problem has been fixed in version 1.4.3-2woody2 for the current stable distribution (woody) and in version 1.4.3-3 for the unstable distribution (sid).  The old stable distribution (potato) is not affected by this problem.
